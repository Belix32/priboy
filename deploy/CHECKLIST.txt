ШАГ 1 — Supabase (SQL Editor)
-----------------------------
1. Откройте SQL Editor (ссылка выше)
2. Скопируйте ВЕСЬ файл: supabase/DEPLOY_ALL.sql
3. Вставьте и нажмите Run
4. Должно появиться: status = "Прибой v2 deployed OK"

-- Supabase Auth (ОБЯЗАТЕЛЬНО для регистрации):
--   Authentication → Providers → Email
--     ✓ Enable Email provider — ВКЛ
--     ✓ Allow new users to sign up — ВКЛ
--     ✗ Confirm email — ВЫКЛ (вход сразу после регистрации)
--   Authentication → Settings
--     ✓ Allow new users to sign up — ВКЛ
--
-- Если отключить Email provider или sign up — на сайте будет
-- «Регистрация отключена в Supabase».


ШАГ 2 — Vercel (Environment Variables)
--------------------------------------
Файл deploy/vercel-env.txt — скопируйте 3 переменные:

  VITE_SUPABASE_URL
  VITE_SUPABASE_ANON_KEY
  VITE_ADMIN_EMAIL

Server-side (Production only, НЕ VITE_*):
  SUPABASE_URL              (= тот же URL что VITE_SUPABASE_URL)
  SUPABASE_SERVICE_ROLE_KEY
  YOOKASSA_SHOP_ID
  YOOKASSA_SECRET_KEY

ЮKassa webhook URL:
  https://priboy-six.vercel.app/api/payments/webhook

Миграция 006 (если DEPLOY_ALL уже был раньше):
  supabase/migrations/006_sprint1_hardening.sql

Vercel → Settings → Environment Variables → Add
Выберите: Production + Preview + Development

После добавления: Deployments → Redeploy


ШАГ 3 — GitHub (запушить v2 код)
---------------------------------
Локально есть незакоммиченные изменения v2.
Нужен push в main, чтобы Vercel собрал новую версию.

  git add .
  git commit -m "feat: Прибой v2 — Supabase, landing, booking, admin"
  git push origin main


ШАГ 4 — Первый вход как admin
------------------------------
1. Зарегистрируйтесь на сайте с email = admin@priboi.ru
   (или измените VITE_ADMIN_EMAIL и app_settings.admin_email на свой email)
2. Войдите — роль admin назначится автоматически (sync_admin_role)
3. Откройте /admin
